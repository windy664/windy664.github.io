做了这么久 WindyAgent，某天我盯着它忽然意识到一件有点尴尬的事：我做的其实是个**很强的被动助手**——你 `!ai` 一句它答一句、你让它干啥它干啥。但「运维 Agent」这五个字里，最值钱的恰恰是**主动**。运维不是等人来问，是自己盯着、出事自己先知道。

这篇记一下我怎么把它从「问答机」往「会自己巡逻的 Agent」推的第一步，以及推的过程中想清楚的几条边界。

<!-- more -->

## 被动和主动，差的不是功能，是「谁先开口」

被动助手的全部交互是：人开口 → 它响应。它再聪明，也得等你。

主动 Agent 多了一条回路：**它自己定时去看 → 发现异常 → 自己开口**。我给它搭的最小闭环就四步：

> 监控（定时巡检子服 TPS / 内存 / 在线 / 掉线）→ 评估（阈值翻转判定）→ 出建议（异常时调 LLM 给一句话诊断 + 处置建议）→ 通知（控制台 + 网页告警）。

听起来简单，但「让它自己动」这件事，每一步都逼我做一个不太舒服的决定。

## 决定一：边沿触发，不然它会把你淹死

第一版我写成「每轮巡检，发现 TPS 低于阈值就报一次」。脑子里跑一遍就知道完蛋：一台卡了的服，30 秒一轮，它会**每 30 秒报一次「卡顿」**，一晚上几百条，告警栏直接变垃圾场。告警一旦泛滥，人就不看了——这比没有告警还糟。

改成**边沿触发**：只在状态**翻转**时报一次。TPS 掉下阈值，报一条「开始卡顿」；恢复了，报一条「已恢复」；中间这段一声不吭。代码上就是给每台服维护一个「当前活跃告警」集合，`add` 成功（从无到有）才推送：

```kotlin
private fun raise(s: String, k: IncidentKind, ...) {
    if (active(s).add(k)) onIncident(...)   // 只有新增才报，重复 add 返回 false
}
```

一个朴素的道理，但做监控的第一天就得想明白：**告警的价值不在「报得全」，在「报得准、不刷屏」**。

## 决定二：它可以「提议」，但先别让它「自己动手」

这是我纠结最久的。一个主动 Agent，发现服务器卡了，要不要让它自己去 `/restart`、去踢人？

技术上不难。但我对着一台**自己从没压力测试过的服**，实在不敢给一个无人盯着的 AI 发「破坏性命令」的方向盘。万一它判断错了，半夜把服重启了，玩家全掉线，我还在睡觉。

所以我定的策略是**「建议，不执行」**：哨兵发现问题，调便宜模型出一句诊断 + 建议处置（甚至给出具体命令），但**只是推给我看**，执行/审批留给人。真要它自动做，高危的也得走我那套人工审批闸。

这条边界我现在越想越觉得对：**自动化的程度，应该匹配你对它判断力的信任程度**。信任是慢慢喂数据喂出来的，不是一上来就把钥匙全给它。先让它当个靠谱的「眼睛 + 参谋」，等它的判断被验证够多次，再逐步放权到「手」。

## 决定三：无人值守时，高危一律拦死

后来我加了条「夜间自动整理知识库」的定时任务——让 Agent 凌晨自己把当天数据提炼进知识库。这就触到一个新问题：**无人值守状态下，Agent 还是带着工具的**，万一它整理着整理着「灵机一动」跑了个危险命令呢？

我给请求上下文加了个 `unattended` 标记。无人值守时，高危命令不再「进审批队列等人批」（没人会半夜批），而是**直接拦截 + 记录**。它能读数据、能写知识库、能写记忆，但碰不到破坏性操作——**不是靠它自觉，是那条路被物理封死**。

## 一个让我清醒的 bug：主动监控的前提是「指标真拿得到」

哨兵上线后我才发现，它在我的测试服（Youer，一个 NeoForge 混合端）上**TPS 永远是 -1**。因为 `getTPS()` 是 Paper 的 API，Youer 没有。结果就是：我的「卡顿告警」从一开始就**永远不会触发**——哨兵睁着眼，但那只眼是瞎的。

后来用 NMS 反射读到了真 TPS（这是另一篇的故事）。但这个坑给我提了个醒：

> 做主动监控，第一件事不是写告警逻辑，是确认**你要监控的指标，在你的目标环境里真的拿得到**。别想当然以为 API 都在。一个拿不到数据的监控，比没有监控更危险——因为它给你一种「我在盯着」的虚假安全感。

## 写在最后

被动助手和主动 Agent 的分界，我现在觉得不在「能力多强」，而在三个问题上：

- 它会**不会自己开口**？（监控回路）
- 它开口会不会**把你淹死**？（边沿触发，告警纪律）
- 它能动手到**什么程度**？（信任与权限的匹配，无人值守的硬边界）

我没有一步到位让它全自动——恰恰相反，我让它**先当眼睛和参谋，手攥在我这**。一个会主动巡逻、会判断、会给建议，但动手前先问过你的 Agent，已经比「你问我答」前进了一大步。剩下的放权，等它证明自己配得上。
